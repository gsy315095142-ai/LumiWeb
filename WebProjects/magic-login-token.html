Token 鉴权与请求封装

请求统一封装（requestPromise）

所有 API 请求通过 requestPromise 函数统一处理：

请求头说明
AuthorizationBearer {access_token} 或 Bearer {tem_token}
mz-store-id当前选中的门店 ID（currentStoreId）
content-typeapplication/json

Token 过期处理（401）

当后端返回 401 状态码时，触发以下流程：

弹窗提示

「登录信息已过期，即将退出！」

标记未登录

设置 user_login = false。

重新登录

调用 wxLogin() 静默获取新 Token，然后 reLaunch 首页刷新。

非 401 错误处理

其他网络错误（非 401）：

  • 隐藏 Loading 动画
  • 返回兜底响应：{code: 501, message: '请求错误，重试', data: null}
  • 不中断用户流程，由各页面自行处理错误码
