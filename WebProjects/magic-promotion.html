优惠活动管理

优惠活动是运营侧的核心营销工具，支持创建多种折扣类型的优惠券，管理活动生命周期（启用/禁用/编辑/删除），并统计分析优惠效果。用户端通过扫码或分享链接领取优惠券。

功能总览

管理端三 Tab 架构

优惠活动管理页面（/pages/promotion/promotion）提供三个核心功能区：

➕ 新增优惠活动
  • 填写活动名称（≤15字）
  • 选择触发条件
  • 配置优惠券参数
  • 设置发放次数和每日限制
  • 开关转赠支持
📋 活动列表管理
  • 查看所有活动
  • 编辑未启用的活动
  • 启用/禁用活动
  • 删除活动
  • 配置门店、期限、有效期
  • 按名称搜索活动
📊 优惠活动统计
  • 按日期范围、门店、活动、优惠券名筛选
  • 查看发放/过期/使用数量
  • 查看核销票券数量及对应人数
  • 导出统计 Excel
创建优惠活动

表单配置项

配置项说明校验规则
活动名称活动的显示名称必填，中文字符 ≤ 15 字（英文算 0.5）
触发条件从后端获取的条件选项（如"不限制条件"）必选，下拉选择
活动形式目前仅支持「发放优惠券」默认值
赠送次数每个用户可领取的次数限制从后端获取的次数选项
优惠券名称券的显示名称必填，中文字符 ≤ 6 字（英文算 0.5）
折扣类型折扣比例 / 固定金额折扣 / 满减必选，切换类型时清空折扣值
折扣值具体的折扣数值比例 ≤ 100%；满减时减免 ≤ 满足金额
满足金额满减类型的门槛金额满减时必填
每日限制每天可领取的次数上限从后端获取的限制选项
支持转赠优惠券是否可以转让给他人开关，默认开启

三种折扣类型

折扣比例
  • type = 0
  • 输入百分比（如 80 = 8折）
  • 不可超过 100%
  • 仅整数，不支持小数
固定金额折扣
  • type = 1
  • 直接减免固定金额
  • 最多保留 2 位小数
  • 无满足金额要求
满减
  • type = 2
  • 需同时填写满足金额和减免金额
  • 减免金额不能大于满足金额
  • 最多保留 2 位小数

创建流程

加载页面配置

调用 /mz-admin/promotionPageOpts，获取触发条件、发放次数、折扣类型、每日限制等下拉选项。

填写表单

管理员填写活动名称、选择条件、配置优惠券参数。

表单校验

活动名称和优惠券名称必填、折扣值必须 > 0、满减时减免不能大于满足金额、折扣比例不能超过 100%。

提交保存

调用 /mz-admin/createPromotion（新建）或 /mz-admin/savePromotion（编辑），成功后重置表单。

活动列表管理

列表数据加载

调用 /mz-admin/promotionList，返回所有活动列表。每个活动包含：

字段说明
id活动 ID
name活动名称
status状态（0=未启用/1=已启用）
store_ids适用门店 ID 数组
start_at / end_at活动期限（起止日期时间）
valid_type优惠券有效期类型（0=固定日期，1=固定天数）
valid_from / valid_to优惠券有效期（固定日期模式）
valid_days领取后有效天数（固定天数模式）

搜索功能

支持按活动名称搜索：

  • 输入关键词后过滤 originalPromotionLists（原始数据备份）
  • 空关键词时恢复显示全部
  • 清除按钮一键恢复

列表配置项（每个活动卡片）

每个活动卡片可独立配置以下参数（仅未启用状态可编辑）：

门店选择
  • 多选门店（store_ids）
  • 从本地存储的 placeLists 获取门店列表
  • 必选至少一个门店
活动期限
  • 开始时间 start_at
  • 结束时间 end_at
  • 结束不能早于开始
  • 自定义日期时间选择器（picker-view）
优惠券有效期
  • 固定日期（valid_type=0）：设置 valid_from 和 valid_to
  • 固定天数（valid_type=1）：设置领取后 N 天有效（valid_days）
  • 点击切换有效期类型，数据联动清空

启用 / 禁用

启用前进行严格的校验：

前置校验

① 至少选择一个门店 → ② 活动期限起止时间已填 → ③ 优惠券有效期已配置（固定日期需起止时间，固定天数需 > 0 天）。

二次确认

弹窗「确定要启用/禁用当前活动吗？」

提交更新

调用 /mz-admin/updatePromotionStatus，传入完整配置：id、status、store_ids、活动期限、有效期类型和值。

编辑活动

  • 仅 status=0（未启用）的活动可编辑，已启用时提示「已启用，无法编辑！」
  • 调用 /mz-admin/promotionDetail 获取活动详情
  • 回填表单数据，切换到「新增」Tab（promotionId > 0 表示编辑模式）
  • 保存时调用 /mz-admin/savePromotion（带 id 参数）

删除活动

  • 仅 status=0 的活动可删除
  • 弹窗确认「是否删除该优惠活动？（若有正在进行中的活动，请先关闭）」
  • 调用 /mz-admin/deletePromotion，传入 id
  • 成功后刷新活动列表
活动统计

筛选条件

统计 Tab 提供多维度筛选：

筛选维度说明
日期范围选择起止日期（默认截止到当月最后一天）
门店下拉选择门店，含「全部」选项
活动下拉选择具体优惠活动，含「全部」选项
优惠券名选择活动后可进一步选择券名

统计指标

调用 /mz-admin/promotionStats，返回以下核心数据：

指标字段
优惠券发放数量promo_code_total
优惠券过期数量promo_code_expired_total
优惠券使用数量promo_code_used_total
优惠券使用对应票券数量coupon_total
票券已核销且未退票数量coupon_used_total
票券已核销且未退票对应人数coupon_used_value_total

导出 Excel

点击导出按钮：

  • 拼接当前筛选条件为 URL 参数
  • 调用 /mz-admin/downloadPromoExcel 下载统计 Excel
  • 文件名格式：优惠活动统计-{开始日期}-{结束日期}.xlsx
  • 同一天时文件名简化为 优惠活动统计-{日期}.xlsx
用户端 — 优惠券领取与查看

优惠券列表（coupons 页）

用户可在优惠券列表页（/pages/coupons/coupons）查看自己的所有优惠券：

  • 加载方式：分页加载，每页 10 条，触底自动加载更多
  • 接口：/mz/getCouponList，传入 page
  • 状态：待使用（status=0）、待核销、已核销、已失效
  • 剩余天数：自动计算 remainingDays（距离 expire_at）
  • 两种模式：普通模式（所有状态）和「待使用票券」模式（仅 status=0）
  • 进入页面前检查 user_login，未登录时弹出授权弹窗

优惠券领取方式

用户可通过以下方式获取优惠券：

📱 扫码领取
  • 扫描二维码进入 vouchersShareJump 页
  • 解析 URL 参数获取券码
  • 弹窗确认后调用兑换接口
🔗 分享链接领取
  • 通过微信分享链接进入
  • 携带 share_code 参数
  • 调用领取接口

领取流程（vouchersShareJump 页）

解析入口参数

支持两种入口：① 小程序路径直接带参数（qrcode、name）；② 扫描二维码带 q 参数（URL 编码），需 decodingFunc 解码。

门店切换

如果参数含 switch_store_id，自动切换到对应门店并跳过定位检查。

等待登录完成

轮询检查 user_info（每 300ms），确保静默登录完成后再操作。

确认领取弹窗

弹窗展示「检测您可获得 1 张【{券名}】，是否立即领取？」。团票则提示「领取后，您将成为团队负责人」。

登录检查

如果用户未主动登录（user_login=false），弹出手机号授权弹窗。

提交兑换

根据券类型调用不同接口，成功后跳转票券列表。

兑换接口路由

券类型接口参数
普通券码/mz/exchangeCoupon{code: qrcode}
分享券/mz/receiveCoupon{code: qrcode}
团票 — 加入/mz/joinGroup{group_code: qrcode}
团票 — 转赠接收/mz/receiveTransferredGroup{group_code, group_transfer_by}

扫码签到入口

如果扫码参数含 check_in=1，直接跳转到签到页（/pages/code/code），不进入券领取流程。

自定义日期时间选择器

选择器设计

活动列表中使用自定义 picker-view 选择器设置时间，支持：

  • 五列滚动：年、月、日、时、分
  • 动态天数：根据选中年月自动计算当月天数（处理 28/29/30/31 天）
  • 防越界：月份切换时自动调整日期索引防止越界
  • 时间校验：关闭选择器时校验：开始时间 ≤ 结束时间
  • 用于设置活动期限（start_at、end_at）和优惠券有效期（valid_from、valid_to）
  • 已启用的活动（status ≠ 0）点击时间字段不弹出选择器
