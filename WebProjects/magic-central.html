模块 08管理端

中控台

中控台是门店运营的核心管理界面，提供场次管理、签到核销、游戏监控、团队批量预约、查询和系统设置等功能。仅拥有门店管理权限的员工/管理员可访问。

功能总览

双模式切换

中控台通过顶部 Tab 切换两个核心模式：

📋 预约管理（book）
  • 按日期查看所有场次排期
  • A/B 厅切换
  • 展开场次详情（角色列表）
  • 签到、创角、电话联系
  • 转入后厅操作
  • 团队批量预约
🎮 游戏监控（monitor）
  • 查看当日进行中的游戏场次
  • 启动游戏（status 1 → 2）
  • 结束游戏（status 2/3 → 5）
  • 实时状态展示

权限控制

中控台通过 getUserPermissions('ticket') 检查当前用户是否拥有票券管理权限。无权限的功能按钮会被隐藏或禁用。

场次排期与状态

数据加载

调用 /mz-admin/getGameRoundsOrdering，传入 date（日期），返回：

字段说明
game_round_lists按厅 ID 分组的场次列表（每个场次含时间、人数、角色、状态等）
store_rooms按门店 ID 分组的厅列表（A 厅、B 厅等）

场次状态流转

status 0 — 未开始 可操作

可签到、创角、编辑角色名/设备号。若已有人预约显示「已预约」，否则显示空白。

status 1/2 — 转后厅 进行中

已核销票券并转入后厅体验，不可再编辑角色。

status 3/4 — 游戏中 进行中

游戏进程已启动，玩家正在体验中。

status 5 — 已完成 结束

游戏已结束，场次归档。

status 6 — 已过期 过期

预约时间已过但未核销，场次自动过期。

A/B 厅切换

场次按厅（store_room）分组展示。顶部显示各厅选项卡，点击切换查看不同厅的场次排期。当前活跃厅索引存储在 activeDateIndex。

智能滚动定位

加载场次列表后，自动根据当前时间滚动到最近的场次位置。计算逻辑：将所有场次开始时间转换为分钟数，找到最后一个不超过当前时间的场次，页面自动 ScrollTo 该位置。仅在查看「今天」的日期时启用。

角色管理

展开场次详情

点击场次行展开该场次下的所有角色（订单）。每个角色展示：

字段说明
nickname角色昵称
gender性别（0=女，1=男）
device_id绑定的设备编号
is_checked_in是否已签到
group_name所属团队名
actived_phone激活者手机号
order_no订单编号

创建角色（形象弹窗）

点击未创建的角色（status=0），弹出形象选择弹窗：

  • 选择性别形象：左/右箭头切换男女角色形象图
  • 填写角色名称：手动输入或点击随机生成（如「聪慧的小狐狸」「勇敢的小老虎」等）
  • 填写设备编号：可选填，支持一键标记为「无」
  • 提交：调用 /mz-admin/updateGameRoleInfo，传入 role_id、nickname、gender
  • 角色名中文不超过 6 个字符

场次统计指标

每个场次行显示以下汇总数据：

页面顶部汇总
  • 已预约场次数 / 总场次数
  • 已预约人数 / 总可容纳人数
每场次的统计
  • 预约人数（bookedNum）
  • 已创建角色数（roleNum）
  • 已签到人数（checkedNum）
  • 已绑定设备数（deviceNum）
签到与电话联系

手动签到

管理员在场次详情中点击签到按钮：

  • 仅对 is_checked_in = false 且 status = 0 的角色生效
  • 弹出确认框「是否对此订单进行签到」
  • 调用 /mz-admin/makeOrderCheckIn，传入 order_id
  • 签到成功后刷新场次列表

电话联系

管理员可直接拨打用户电话：

  • 如果角色已有 actived_phone，直接拨打
  • 否则调用 /mz-admin/getPhoneByOrder，传入 order_id 获取手机号
  • 获取后缓存到 orderPhone 对象中，避免重复请求
  • 使用 wx.makePhoneCall 发起电话
转入后厅

转入流程

管理员将已签到的玩家组转入后厅体验区：

前置检查

角色创建数 ≥ 已签到数、签到数 > 0、已输入场次号码、已选择目标厅。

选择目标厅

弹窗展示所有已开启的厅，每个厅显示其同时间段场次的可转入状态。系统自动计算：同组可合并、空厅可迁入、已满/状态不对则不可选。

确认转入

如果目标厅有未签到的用户，弹二次确认；否则弹「确认核销并引导体验」。

提交

调用 /mz-admin/gameRoundEnterScene，传入 round_id、number_plate、new_round_id。

目标厅可转判断规则

源组 → 目标组条件
同组（group_id 相同）目标厅预约人数 + 源人数 ≤ 上限 且 目标 status=0
有组 → 空组目标厅预约人数=0 且 上限 ≥ 源人数 且 status=0
空组 → 空组目标厅预约人数 + 源人数 ≤ 上限 且 status=0
不同组不可转入（canSelected = false）

场次号码

转入后厅时需输入 number_plate（场次号码/号码牌），用于标识该组的排队或编号。可在转入弹窗中直接编辑。

游戏监控

数据加载

切换到「游戏监控」Tab 时，调用 /mz-admin/getPlayingGameRoundList，传入 date，返回当日所有进行中的游戏场次。

监控列表展示

每条记录显示：

  • 时间段：time_start - time_end
  • 状态：1=准备中、2=启动中、3=游戏中、4=待结束、5=已完成
  • 角色列表：该场次下的所有玩家角色信息
  • 统计：总场次数、总参与人数

启动游戏

对 status=1（准备中）的场次：

  • 点击启动按钮，弹出确认框「确认启动该组玩家的进程？」
  • 调用 /mz-admin/startGame，传入 round_id
  • 成功后刷新监控列表

结束游戏

对 status=2/3（启动中/游戏中）的场次：

  • 点击结束按钮，弹出确认框「确认结束该组玩家的进程？」
  • 调用 /mz-admin/endGame，传入 round_id
  • 成功后刷新监控列表
团队批量预约

导入团队文件

管理员可从 Excel 文件批量导入团队预约数据：

选择文件

调用 wx.chooseMessageFile，仅允许 .xlsx / .xls 格式。

解析文件

调用 wxFileUpload 上传到 /mz-admin/getGroupFile 接口，后端解析返回成员列表 memberTeam 和错误行数 error_line。

选择日期和时段

选择预约日期 → 调用 /mz/getGameRoundList 获取可用时段 → 选择起始时段，系统自动计算结束时段（按团队人数 / 4 向上取整确定所需连续场次数）。

提交预约

调用 /mz-admin/orderGameRoundByGroup，传入日期、起始/结束 round_id、玩家列表。

撤销团队场次

调用 /mz-admin/cancelGroupRound，传入 round_id，二次确认后撤销整个团队预约。

模板下载与分享

提供团队预约模板文件下载（cdn.lixiaoqian.com/magic-space/a_teamfile.xlsx），通过 wx.downloadFile + wx.shareFileMessage 以文件消息形式分享给用户填写。

票券 / 订单查询（query 页）

三种查询方式

Tab查询方式后端 method
券码查询输入官方或第三方渠道的券码coupon
手机号查询输入手机号查询该用户所有票券mobile
订单号查询输入订单号查询关联票券order

查询流程

  • 所有查询统一调用 /mz-admin/searchCoupon，传入 method 和 search_input
  • 支持粘贴按钮（从剪贴板快速填入）
  • 查询结果包含票券状态、场次信息、渠道信息、团票子场次等
  • 普通角色（current_role=0）只能使用手机号查询，券码和订单号 Tab 被隐藏

结果展示

查询结果按渠道分组，每组包含：

  • 票券卡片：状态、券码、类型、有效期、所属门店、兑换/使用时间、发票状态
  • 团票展开：显示各子场次的时间、地点、预约人数、联系电话
  • 场次卡片：场次时间、地点、预约人数
  • 转让标记：已赠出/已获赠的票券有特殊标记
预约确认页（complete 页）

页面用途

用户预约成功后跳转到的确认页面，也支持通过分享链接进入。展示预约详情并提供操作入口。

预约详情展示

信息说明
预约日期场次日期
时段场次time_start - time_end
预定人数已预约人数
联系电话可修改（仅团长）
门店信息门店名称和地址

可用操作

团长（is_owner=1）
  • 创建角色
  • 撤销预约（进入后厅前）
  • 改签（修改场次时间）
  • 修改预定人数
  • 修改联系电话
  • 分享激活码给团员
  • 扫码签到
团员
  • 创建角色
  • 查看预约详情
  • 扫码签到
  • 分享邀请

分享功能

预约确认页支持微信分享，分享时：

  • 生成分享图：Canvas 绘制含门店地址、日期、时段、人数、电话的分享卡片
  • 分享标题：「一封来自魔法学院的入学邀请函」
  • 分享路径：带 order_id、status、share 参数的 complete 页路径
  • 切换门店：分享链接带 switch_store_id，打开后自动切换门店

改签逻辑

  • 仅团长可操作，且有剩余改签次数（reorder_times ≥ 1）
  • 场次状态必须 ≤ 1（未进入后厅）
  • 团队预约（group_id > 0）不支持改签
  • 点击改签跳转到预约页面（带 order_id 和 orderDate 参数）

修改人数

  • 弹出选择器（1/2/3/4 人），当前人数高亮
  • 新人数不能超过已使用票券总人数上限（coupon_value_sum）
  • 调用 /mz/changeOrderPlayerNum，传入 order_id 和新人数
  • 人数减少无需修改，现场告知即可（弹窗提示）
系统设置

设置弹窗

点击设置按钮打开弹窗，包含两个 Tab：

场次与票价设置
  • 选择门店
  • 设置起止日期
  • 是否开启预约
  • 设置营业时间段
  • 场次间隔时间（分钟）
  • 票价设置（1-4人券）
调整开放预约
  • 按门店查看所有厅
  • 开启/关闭各厅预约
  • 实时切换厅的 status

场次与票价设置流程

加载数据

打开弹窗时调用 /mz-admin/getRoundPriceConf，传入日期范围和门店 ID，回显当前配置。

编辑配置

修改日期、门店、开关、时间、间隔、票价等。修改日期/门店时自动重新加载配置。

表单校验

必须选择门店；开启状态时必须填写时间段和间隔；日期不能早于今天；结束日期不能早于开始日期。

二次确认并保存

弹出确认弹窗展示修改内容。调用 /mz-admin/saveRoundPriceConf 提交。

开放预约切换

对每个厅的预约状态单独控制：

  • 调用 /mz-admin/saveStoreRoomStatus，传入 store_room_id 和 status
  • status=1 表示开启，status=2 表示关闭
  • 切换前弹出确认框「确定打开/关闭 xxx？」
  • 切换后刷新列表状态
