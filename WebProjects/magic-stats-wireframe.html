📊 汇总统计 — totalStatData

接口 /mz-admin/getStoreDataStats 返回的 data.totalStatData，汇总所有票券类型。

  • name = "汇总统计" — 卡片标题，蓝竖线 border-left:10rpx solid #007bff
  • nickname = "总订场票券" — 渐变区域标题
  • ticketNum（数量）：选定日期范围内的总订场票券张数（包含所有类型：酒店订场 + 团购，包含已使用和未使用的票券，不包含已退票和已过期）
  • ticketValue（人数）：选定日期范围内的总订场人数（每张票券对应的人数之和）
  • refundNum（已退票数量）：选定日期范围内已退票的票券张数（退票申请已通过的票券）
  • overdueNum（已过期数量）：选定日期范围内已过期的票券张数（超过有效期未使用的票券）
对账关系：生成总票券 = ticketNum + refundNum + overdueNum（正常使用 + 退票 + 过期 = 全部生成的票券）

📋 酒店票券统计 — newStatsData[0]

每个分项卡片结构相同，由后端 newStatsData[] 数组驱动。字段按条件显示（wx:if）。

  • name = "酒店票券统计" — 卡片标题
  • nickname = "酒店订场票券" — 渐变区标题
  • ticketNum：酒店订场票券张数（用户通过酒店直接预约/购票生成的票券）
  • ticketValue：酒店订场人数（票券面值之和）
  • receiveNum（已领取票券）：已被用户领取（关联到 coupon）的票券数量
  • receiveNoOrderNum（已领未下单）：已领取但尚未创建预约订单的票券数量（领了券但还没约场次）
  • refundNum（已退票数量）：退票申请已通过的票券张数
  • overdueNum（已过期数量）：超过有效期未使用的票券张数
  • qrcodeNum（二维码已扫码）：通过二维码扫码方式核销/使用的票券数
  • qrcodeInvalidNum（二维码已失效）：二维码已超过有效期或已被作废的票券数
  • packages[]：套餐包明细数据，展开后显示每个套餐的名称、数量、人数

👥 团购票统计（展开状态）— newStatsData[1+]

团购票的分项统计，额外包含 packageRefundNum 和 packageNum 两个套餐指标。

  • ticketNum：团购渠道生成的票券张数（通过团购码兑换生成的票券）
  • ticketValue：团购人数（团购票面值之和）
  • packageRefundNum（套餐退票数量）：团购套餐中退票的票券张数（团购票券的特殊退票统计，与 refundNum 可能不同）
  • packageNum（套餐总数量）：团购套餐生成的总票券张数（包含所有状态的团购票券）
  • packages[] 套餐包明细：
    • name：套餐名称（如"基础团购套餐（3人）"）
    • num：该套餐的票券张数
    • value：该套餐的人数
  • 展开/收起：handleShowPackage 切换 expanded，箭头图标 rotate-180。无数据时显示「暂无数据」
  • packageRefundNum 大值：当值 >100000 时，数字缩小字号（statistics-num-mini）
数据流程 & API & 字段定义

数据加载流程

步骤触发说明
1. 初始化onShow轮询读取 placeLists（100ms 间隔），酒店列表首位插入「当前选择：全部」（id=0）
2. 日期快捷search-date 组件 attacheddataIndex=3 默认选中「本月」，自动计算起止日期并触发查询
3. 选日期onTicketDateChange手动选起止日期，结束日期 ≥ 开始日期，两个日期都填满后自动触发 onSearch
4. 选酒店onHotelChange更新 selectedHotel，触发 handleGetStatData
5. 请求数据handleGetStatDataPOST /mz-admin/getStoreDataStats {date_start, date_end, store_id}
6. 权限判断无 view-stat-hotelstoreId 取 getItem('currentStoreId')（当前用户所属门店），酒店选择器不显示
7. 渲染接口返回 code=0设置 totalStatData + newStatsData + dataLoaded=true
8. 展开/收起handleShowPackage切换 expanded，显示 packages[] 套餐包明细
9. 导出handleExport仅 view-stat-hotel + selectedHotel > 0 时可见，调用 wxFileDownload

API 接口

接口方法参数用途
/mz-admin/getStoreDataStatsPOST{date_start, date_end, store_id}获取统计卡片数据（汇总 + 分项）
/mz-admin/downloadStatementsGETdate_start & date_end & store_id导出对账单 Excel，文件名 {storeName}对账单-{start}至{end}.xlsx

totalStatData（汇总统计）字段定义

字段类型含义对账说明
nameString卡片标题：固定 "汇总统计"—
nicknameString主数据行标题：固定 "总订场票券"—
ticketNumNumber总订场票券张数（选定日期范围内的有效票券总量）正常使用中的票券
ticketValueNumber总订场人数（每张票券对应的人数之和）实际到场人次
refundNum{value, label}已退票数量退票申请已通过的票券
overdueNum{value, label}已过期数量超过有效期未使用的票券
对账公式：生成总票券 = ticketNum + refundNum.value + overdueNum.value
即：正常使用的票 + 已退票 + 已过期 = 该时段内生成的全部票券

newStatsData[]（分项统计）字段定义

字段类型含义显示条件
nameString卡片标题（如"酒店票券统计"、"团购票统计"）始终
nicknameString主数据行标题（如"酒店订场票券"、"团购订场票券"）始终
ticketNumNumber该类别的票券张数始终
ticketValueNumber该类别的人数始终
receiveNum{value, label}已领取票券：已被用户领取/绑定到 coupon 的票券字段存在时
receiveNoOrderNum{value, label}已领未下单：已领取但尚未创建预约订单的票券（领了券还没约场次）字段存在时
refundNum{value, label}已退票数量：退票申请已通过的票券张数字段存在时
overdueNum{value, label}已过期数量：超过有效期未使用的票券张数字段存在时
qrcodeNum{value, label}二维码已扫码：通过二维码扫码方式核销/使用的票券数字段存在时
qrcodeInvalidNum{value, label}二维码已失效：二维码超过有效期或被作废的票券数字段存在时
packageRefundNum{value, label}套餐退票数量：团购套餐中的退票票券数（value>100000 时缩小字号）字段存在时（团购）
packageNum{value, label}套餐总数量：团购套餐生成的总票券张数字段存在时（团购）
packages[]Array套餐包明细（name/num/value），展开/收起显示字段存在时
expandedBoolean套餐包列表展开状态，默认 false前端控制

packages[] 套餐包明细字段

字段含义
name套餐名称（如"基础团购套餐（3人）"、"豪华团购套餐（5人）"）
num该套餐的票券张数
value该套餐的人数（通常等于套餐限定人数 × 张数）
