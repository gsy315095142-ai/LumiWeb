🔵 通用字段（酒店 & 团购都有）
ticketNum 该渠道正常使用中的票券张数
ticketValue 该渠道的到场人数
receiveNum 已被用户领取/绑定到 coupon 的票券
receiveNoOrderNum 领了券但还没约场次的票券
🟠 异常/退出指标
refundNum 退票已通过的票券
overdueNum 超期未使用的票券
🟣 二维码指标
qrcodeNum 通过扫码核销的票券
qrcodeInvalidNum 二维码过期或作废的票券
🔶 团购专属（虚线框）
packageRefundNum 团购套餐退票数
packageNum 团购套餐总生成数
packages[] 展开后每个套餐的 name/num/value
📐 对账关系
ticketNum（正常）+ refundNum（退票）+ overdueNum（过期）= 该渠道总生成票券
receiveNum ≥ receiveNoOrderNum（领了才能不下单）
qrcodeNum + qrcodeInvalidNum = 二维码渠道的总票券数

数据流程 & API & 权限

数据加载流程

步骤触发说明
1. 初始化onShow轮询 placeLists（100ms），酒店列表首位插入「全部」（id=0）
2. 默认日期search-date attacheddataIndex=3 默认「本月」，自动计算起止并触发查询
3. 手动选日期onTicketDateChange选满两个日期自动触发 onSearch
4. 选酒店onHotelChange更新 selectedHotel → handleGetStatData
5. 请求handleGetStatDataPOST /mz-admin/getStoreDataStats
6. 渲染返回 code=0设置 totalStatData + newStatsData + dataLoaded
7. 展开handleShowPackage切换 expanded 显示 packages[]
8. 导出handleExport仅非全部酒店时可见，wxFileDownload 导出 Excel

API 接口

接口方法参数用途
/mz-admin/getStoreDataStatsPOST{date_start, date_end, store_id}统计卡片数据（汇总 + 分项）
/mz-admin/downloadStatementsGETdate_start & date_end & store_id导出对账单 Excel

权限说明

权限 Key适用角色可见内容
view-stat-hotel管理员（current_role=1）酒店选择器 + 导出按钮，可查所有门店
create-stat-account管理员（current_role=1）生成对账单（导出功能）
无 view-stat-hotel门店员工自动用 currentStoreId 查本店，不显示选择器
