团队预约管理

团队预约管理（团票占位系统）允许团长和团员通过可视化网格完成自助选座、创建角色、换位调整。管理端可通过中控台批量导入团队或在场次详情中管理团队。

概念与入口

什么是团票

团票（GROUP_TICKET）是一种支持多人同时预约的特殊票券类型。购买后获得一个 group_code，团长和团员通过该码进入占位页面，自助分配各场次的角色位置。

入口方式

入口说明
小程序页面跳转 /pages/groupTeam/groupTeam?group_code=xxx
扫码扫描团票二维码 → vouchersShareJump → 携带 group_code 跳转
分享链接团队成员通过微信分享链接进入
中控台管理端场次详情中查看团队占位
页面布局

页面结构

团票占位页面由以下区域组成：

  • 顶部 Banner：展示背景图 + 所属门店名称
  • 预约信息栏：预约日期、星期
  • 统计栏：团队已预约人数 / 团票上限人数
  • 场次-角色网格：核心占位区域（行=场次，列=角色位）
  • 底部操作栏：占位调整按钮 / 换位步骤提示
  • 悬浮刷新按钮：右下角一键刷新数据
  • 角色编辑弹窗：点击格子后弹出的全屏蒙层表单

角色位补齐逻辑

每场次的格子数固定为 4 列，但实际可占位数量由 player_max（1-4）决定：

  • 已有角色：从后端返回的真实角色数据
  • 虚拟占位：补足到 player_max 个可点击的空位（id 格式 virtual-{round_id}-{index}）
  • 空白格：player_max 不足 4 时，剩余位置显示为空白（不可点击）
  • player_max 小于 1 时自动修正为 1，大于 4 时修正为 4
权限体系

三种角色身份

身份判断条件权限
团队负责人 groupMaster == userInfo.id 可占位、可编辑所有角色、可换位调整、可删除任何角色
本门店店员 isStoreStaffOfThisStore = true
（store_staff 中有该门店的 staff_role）		 等同于团队负责人权限
普通团员 非负责人且非店员 仅可占空位、编辑自己创建的角色、不可换位调整

操作权限矩阵

操作负责人店员团员

删除角色✅（所有）✅（所有）✅（仅自己创建的）

权限校验时机

页面加载时（initData）自动计算：

  • 从 store_staff 中查找与团票门店匹配的员工记录
  • 有 staff_role → isStoreStaffOfThisStore = true
  • 与 groupMaster 比较 → 是否为团队负责人
  • 综合判断：isAdmin = (groupMaster == userInfo.id) || isStoreStaffOfThisStore
占位操作（创建/编辑角色）

点击格子触发占位

点击网格中的角色格子（handleGetPos）：

场次状态检查

若 round.status > 0（已开启），提示「该场次已开启，无法修改/新增角色」。

所有者检查

若已有角色（created_by ≠ 0）且非本人创建且非管理员，提示「您不是该位置所有者，不能操作」。

弹出角色编辑弹窗

展示角色名称、性别、联系方式输入框。编辑已有角色时回填数据。

保存占位

校验通过后调用 /mz/reserveGroupSeat。

角色编辑弹窗

全屏蒙层弹窗，包含以下字段：

字段说明校验
角色名称文本输入（type=nickname），微信自动推荐必填，≤6 字，不可含 Emoji
性别女/男切换按钮，点击高亮切换默认 0（女），可选 1（男）
联系方式数字键盘输入手机号必填，必须是 11 位数字
💡 智能预填

若角色联系方式为空且当前用户不是管理员，自动填入登录用户的手机号。

保存占位接口

调用 /mz/reserveGroupSeat（POST），参数：

参数说明
group_code团票码
round_id场次 ID
role_nickname角色名称
role_gender性别（0/1）
role_phone联系电话
role_id（可选）编辑已有角色时传入

删除角色

在角色编辑弹窗中点击「删除角色」：

  • 仅已创建的角色（created_by ≠ 0）显示删除按钮
  • 弹窗确认「确认取消该角色的归属吗？」
  • 调用 /mz/deleteGroupSeat，传入 group_code 和 role_id
  • 成功后关闭弹窗并刷新网格
换位调整

换位功能说明

团队负责人和门店店员可将角色在不同位置之间调换，用于优化团队分组。仅管理员可操作。

换位流程（三步操作）

Step 0 → 点击「占位调整」

底部出现「占位调整」按钮（仅负责人/店员可见）。点击进入换位模式，switchStep = 1。

Step 1 → 选择源角色 请选择需要换位的角色

点击网格中一个已有角色（created_by ≠ 0）作为源位置。点击空位提示「该位置未有选择」。选中后高亮显示（currentRoleId 匹配），switchStep = 2。

Step 2 → 选择目标位置 请选择交换的位置

点击另一个格子作为目标（可以是已有角色或空位）。弹窗确认「是否交换位置？」。

提交换位

调用 /mz/adjustGroupSeat（POST），传入 group_code、role_id（源）、target_role_id 或 target_round_id（目标）。

换位参数路由

目标位置类型参数说明
目标为已有角色target_role_id两个角色互换位置
目标为空位（虚拟位）target_round_id角色移到该场次的空位
目标异常—提示「目标位置异常」并取消操作

取消换位

换位过程中随时可点击「取消」按钮，重置 switchStep = 0 并清除选中状态。

数据初始化流程

页面加载流程

获取 group_code

从页面参数中获取 group_code，为空则直接返回上一页。

等待登录

轮询 user_info（每 300ms），确保静默登录完成。未主动登录则弹出手机号授权。

加载占位数据

调用 /mz/getGroupSeatData（POST），传入 group_code。

数据组装

后端返回 rounds（场次列表）和 roles（角色列表）。前端按 round_id 分组，补齐虚拟位和空白格。

页面渲染

设置日期、星期、团队人数统计、门店信息、店员权限标记等。

后端返回数据结构

字段类型说明
selectDataString预约日期（YYYY-MM-DD）
roundsArray场次列表（id, time_start, time_end, player_max, status, room_show_name）
rolesArray角色列表（id, round_id, nickname, gender, actived_phone, created_by, status）
groupNumNumber团票可预约总人数上限
groupMasterNumber团队负责人用户 ID
store_nameString所属门店名称
store_idNumber所属门店 ID

时间格式化

场次时间（time_start、time_end）后端返回含秒数（如 10:00:00），前端自动截取为 HH:MM 格式。

错误处理与特殊场景

错误码处理

错误码处理方式
code > 0（通用）Toast 提示后端返回的 message
code = 700数据已失效/过期，Toast 后延迟 2 秒自动跳回首页（reLaunch）
数据为空（!data）Toast「数据为空」后跳回首页

门店自动切换

如果入口参数包含 switch_store_id，页面加载时自动写入 currentStoreId，实现跨门店团票的无感切换。

页面生命周期保护

onHide 和 onUnload 时自动清除登录轮询定时器（setInterval），防止内存泄漏。
