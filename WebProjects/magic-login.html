模块 07

登录与身份验证

登录系统分为两个层次：应用启动时的静默登录（获取 Token），以及用户主动登录（完善资料 + 手机号授权）。整体采用 JWT Token 鉴权，支持 Token 过期自动刷新、多环境切换和全局错误处理。

登录架构总览

应用启动（app.js → onLaunch）

小程序启动时自动调用 wxLogin()，静默换取 Token。用户无感知。

Token 写入本地存储

登录成功后 Token 写入 access_token，同时存储用户信息、门店列表、角色等。

请求自动携带 Token

所有 API 请求通过 requestPromise 统一封装，自动在 Header 中附加 Authorization: Bearer {token}。

用户主动登录（login 页）

需要身份的操作（预约、查看订单等）触发登录页，用户填写头像昵称、授权手机号后完成登录。

静默登录（wxLogin）

触发时机

wxLogin() 在以下场景自动触发：

  • 应用启动：app.js → onLaunch 中调用
  • 登录页加载：login.js → onShow 中调用
  • Token 过期：401 响应后自动调用重新登录
  • 注销账号：注销后重新执行登录流程

登录流程

调用 wx.login() 获取 code

微信 SDK 返回临时登录凭证 code。

发送 code 到后端

调用 /mz/login，后端通过 code 换取用户 OpenID，返回 Token 和用户数据。

写入本地存储

登录成功后批量写入以下数据：

登录返回数据

存储 Key数据内容
user_info用户 ID、头像、昵称、手机号、是否新用户
access_tokenJWT Token（后续所有请求携带）
expire_atToken 过期时间
service_phone默认客服电话
placeLists门店列表（用户可用的门店）
store_staff门店员工角色列表（管理端用）
super_man是否超级管理员（role == 1）
current_role当前用户角色类型

超时与容错机制

wxLogin() 内置了完善的重试机制：

  • 5 秒慢提示：登录超过 5 秒时 Toast 提示「登录较慢，请稍候…」
  • 12 秒超时：登录超过 12 秒弹窗询问「登录请求超时，是否重试？」
  • 重复保护：使用 finished 标志位防止超时回调和正常回调同时执行
  • 失败处理：网络错误时 Toast 提示「登录失败，请检查网络」

清除旧数据

登录时自动清除 placeLists（门店列表），确保每次启动获取最新的门店数据。

用户主动登录（login 页）

登录页面结构

登录页（/pages/login/login）包含以下元素：

  • 背景图：CDN 加载的登录背景大图（cdn.lixiaoqian.com/magic-space/login-bg.jpg）
  • 头像选择：通过微信 chooseAvatar 组件选择头像
  • 昵称输入：通过 input type=nickname 组件输入（微信自动填充建议）
  • 手机号授权按钮：新用户必须授权手机号
  • 进入小程序按钮：老用户可直接进入
  • 协议勾选：须同意《服务条款》和《隐私政策》

双轨登录策略

根据用户是否为新用户（is_new）提供不同的登录路径：

用户类型操作流程
新用户（is_new = true）选择头像 → 输入昵称 → 授权手机号 → 勾选协议 → 进入
老用户（is_new = false）勾选协议 → 点击「进入小程序」→ 自动跳转首页

头像与昵称

微信提供了两种获取用户资料的方式（按基础库版本自动适配）：

  • 新方式：使用 button open-type=chooseAvatar 选择头像 + input type=nickname 输入昵称（基础库 2.21.2+）
  • 旧方式：使用 wx.getUserProfile 弹窗获取头像昵称（已逐步废弃）
  • 通过 canIUseNicknameComp 和 canIUseGetUserProfile 自动判断可用方式

手机号授权

新用户必须授权手机号，流程：

  • 点击「手机号授权」按钮，触发 open-type=getPhoneNumber
  • 微信返回 encryptedData、iv、code
  • 调用 /mz/updatePhone 解密手机号并绑定
  • 成功后 Token 从临时存储（tem_token）转正为 access_token
  • 更新 user_info 中的手机号字段

临时 Token 机制

登录页使用双 Token 策略解决「先登录后授权」的问题：

  • tem_token：静默登录后的临时 Token，用于手机号授权请求
  • access_token：手机号授权成功后转正的正式 Token
  • 点击「进入小程序」时，将 tem_token 转存为 access_token 并清空临时 Token
  • Token 为空时提示错误并自动刷新页面
协议与合规

服务条款 & 隐私政策

登录页底部提供协议勾选框：

  • 默认未勾选（agreeDeal = false）
  • 点击勾选框切换状态
  • 点击「服务条款」或「隐私政策」跳转对应页面
  • 未勾选时点击任何操作按钮均提示「请阅读并同意《服务条款》和《隐私政策》」
Token 鉴权与请求封装

请求统一封装（requestPromise）

所有 API 请求通过 requestPromise 函数统一处理：

请求头说明
AuthorizationBearer {access_token} 或 Bearer {tem_token}
mz-store-id当前选中的门店 ID（currentStoreId）
content-typeapplication/json

Token 过期处理（401）

当后端返回 401 状态码时，触发以下流程：

弹窗提示

「登录信息已过期，即将退出！」

标记未登录

设置 user_login = false。

重新登录

调用 wxLogin() 静默获取新 Token，然后 reLaunch 首页刷新。

非 401 错误处理

其他网络错误（非 401）：

  • 隐藏 Loading 动画
  • 返回兜底响应：{code: 501, message: '请求错误，重试', data: null}
  • 不中断用户流程，由各页面自行处理错误码

文件上传/下载

文件操作也统一携带 Token 和门店 ID：

  • 上传（wxFileUpload）：通过 wx.uploadFile 上传，路径为 baseUrl + /mz-admin/{url}
  • 下载（wxFileDownload）：通过 wx.downloadFile 下载，支持完整 URL 或相对路径
  • 下载后默认通过 wx.shareFileMessage 以文件消息分享
本地存储体系

存储工具函数（storage.js）

基于 wx.setStorageSync / getStorageSync 封装，提供以下功能：

函数说明
setItem(key, value)存储数据，对象/数组自动 JSON 序列化
getItem(key)读取数据，自动尝试 JSON 反序列化（仅对合法 JSON 字符串）

登录态判断与页面守卫

全局登录判断逻辑

各页面通过以下模式判断用户登录状态：

  • 检查 user_login：用户是否已主动完成登录（头像+昵称+手机号）
  • 未登录弹窗：弹出手机号授权组件（open-type=getPhoneNumber）
  • 授权成功：调用 /mz/updatePhone 绑定手机号，标记 user_login = true，刷新页面数据
  • 拒绝授权：用户关闭弹窗后不强制跳转，但功能受限

管理端权限判断

管理端页面额外检查：

  • 员工角色：store_staff 数组中是否有当前门店的管理权限
  • 超级管理员：super_man = true 时拥有所有权限
  • 功能权限：通过 getUserPermissions() 检查具体功能权限
  • 无权限的操作按钮隐藏或提示「无操作权限」
