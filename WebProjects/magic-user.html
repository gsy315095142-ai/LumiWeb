模块 07

用户中心

用户中心是小程序的底部 Tab 页之一，聚合了用户的所有核心数据：我的场次、我的票券、优惠券，以及团购兑换、转赠、撤销场次等操作入口。同时包含设置页的个人资料管理。

页面整体结构

顶部用户信息区

页面顶部展示用户状态信息：

  • 头像 & 昵称：显示微信头像和昵称（未登录时显示默认头像 +「微信用户」）
  • 登录按钮：未登录时显示「登录」按钮，点击弹出手机号授权弹窗
  • 团购兑换入口：已登录用户可点击团购兑换图片，弹出兑换弹窗输入团购码
  • 联系客服：点击拨打当前门店或默认客服电话
  • 设置入口：右上角齿轮图标进入设置页

功能导航区（5 个入口 + 角标）

用户信息下方为功能入口网格，每项带数字角标：

入口角标来源跳转目标
待使用票券coupon_pending_count/pages/coupons/coupons?type=1
我的票券coupon_count票券 Tab
创建角色order_un_role_count/pages/role/role?order={id}
待体验场次order_un_check_count/pages/orders/orders?type=0
我的订单order_count场次 Tab

角标为 0 时点击提示「暂无更多数据」。创建角色入口会自动携带待创角订单 ID。

三个 Tab 内容区

功能导航下方为三个 Tab 切换：

  • 场次（round）：我的场次列表，按待体验/已体验分组
  • 票券（coupon）：我的票券列表，按待使用/其他分组
  • 优惠券（promotion）：我的优惠券列表

可通过外部参数 jumpToTicket 跳转后默认显示票券 Tab。

我的场次

数据加载

调用 /mz/getUserRounds 获取用户所有场次数据（每页 100 条），包含：

  • 场次信息：日期、开始时间、所属门店
  • 订单信息：预约人数、订单号、是否为团购订单
  • 票券信息：关联的票券名称和面值
  • 签到状态：is_checked_in
  • 场次状态：status（0-5）

列表分组与排序

场次按状态分为两组：

待体验 (0) 已激活 (1) 游戏中 (2) 已体验 (3) 已取消 (4) 已过期 (5)
  • 待体验（status < 2）：按日期升序 → 时间升序排列，最近的场次排在前面
  • 已体验（status ≥ 2）：按日期降序 → 时间降序排列，最近的已完成场次排在前面

场次卡片操作

每个场次卡片显示门店名称、日期时间、人数、订单号，并提供以下操作：

  • 点击卡片：跳转完成页（/pages/complete/complete），仅 status < 2 时可跳转
  • 撤销场次：左滑出现撤销按钮（仅 platform=lumi 的票券支持左滑）
    • 开场前 3 小时以上才可撤销
    • 确认弹窗提示：撤销后票券将重置为待使用状态
    • 调用 /mz/cancelGameRoundOrder
  • 复制订单号：点击订单号一键复制

场次状态视觉

不同状态的场次卡片有不同的视觉样式：

状态卡片样式背景图标
待体验默认样式—
已激活默认样式—
游戏中蓝色阴影 + 蓝色背景game 图标
已体验绿色阴影 + 绿色背景expired 图标
已取消灰色阴影 + 灰色背景cancel 图标
已过期灰色阴影 + 灰色背景overdate 图标
我的票券

数据加载

调用 /mz/getCouponList 获取用户所有票券。每张票券包含：

  • 票券名称、面值（value）、所属门店
  • 票券码（code）、平台类型（inner/group）
  • 创建时间、过期时间、剩余天数
  • 使用状态、转让状态（transfer_status）

智能分组逻辑

票券按使用状态分为「待使用」和「其他」两组，分组逻辑较复杂：

类型待使用条件其他
普通票status=0、未过期、未被转让出去status≠0、已过期、已转让
团购票status 为 0/1、未过期、非本人转出已使用、已过期、已转出
转让接收transfer_status=2 且当前用户是接收方—

排序规则：普通票按兑换时间降序，团购票按预约日期降序。

票券状态流转

待使用 (0) 待核销 (1) 已核销 (2) 已过期 (3) 已失效 (4)

票券操作

票券卡片支持多种操作：

  • 一键预约：点击「预约」跳转预约页，携带票券 ID 和面值，自动切换门店
  • 赠送票券：点击「赠送」弹出确认框，确认后通过微信分享给好友
    • 调用 /mz/markCouponTransferable 标记可转让
    • 生成 Canvas 海报（含票券类型、兑换时间、门店）作为分享封面
    • 分享链接：/pages/vouchersShareJump?share_code=1&qrcode={code}
    • 注意：赠送后自己无法再使用
  • 团购票分享：将团购票分享给团队成员预约场次
  • 团购票转赠：将整个团购票转让给他人
    • 调用 /mz/markGroupTransferable 标记可转让
    • 离屏 Canvas 生成分享海报（门店名 + 团队人数 + 预约日期）
  • 退票：左滑出现退票按钮（仅 platform=lumi）
    • 确认弹窗：退掉后不可撤销
    • 调用 /mz/refundCouponWithPayOrder
我的优惠券

优惠券列表

调用 /mz/promoCodeList 获取用户的优惠码列表，每张优惠券显示：

  • 折扣信息：支持三种类型
    • 按比例折扣：如「8折」「6.5折」
    • 固定金额折扣：如「20元折扣」
    • 满减：如「满100减30元」
  • 有效期：显示截止日期 valid_to
  • 使用状态：未使用 / 已使用 / 已过期
  • 所属门店

优惠券赠送

优惠券支持通过微信分享赠送给好友，流程与票券赠送类似：

  • Canvas 生成分享海报（不同背景图 + 优惠券类型 + 截止时间 + 门店）
  • 分享标题：「赠送你一张魔法学院优惠券，请查收~」
团购兑换

兑换入口

用户中心顶部有团购兑换入口（图片按钮），点击后弹出兑换弹窗：

  • 输入团购码（group_code）
  • 调用兑换接口验证团购码有效性
  • 兑换成功后票券自动添加到用户的票券列表
  • 关闭弹窗后自动刷新数据
设置页（settings）

个人资料

设置页（/pages/settings/settings）展示和管理用户的个人信息：

字段说明
头像支持通过 chooseAvatar 组件选择新头像
昵称支持通过 input type=nickname 组件修改
手机号脱敏显示（138****1234），不可直接修改
版本号正式环境显示小程序版本号，开发环境不显示

退出登录

点击退出登录按钮，确认后将 user_login 设为 false，2 秒后跳转首页（reLaunch）。

注销账号

点击注销账号按钮，二次确认后调用 /mz/closeAccount：

  • 确认弹窗内容：「账号注销后将无法恢复，确认继续？」
  • 调用注销接口成功后清除本地用户信息
  • 自动重新执行微信登录流程（wxLogin）
  • 2 秒后跳转首页
左滑交互机制

触发条件

用户在场次/票券列表中左滑可触发操作按钮：

  • 水平阈值：X 轴左滑超过 100px 时激活，右滑超过 20px 时关闭
  • 垂直过滤：Y 轴移动距离 > X 轴时判定为垂直滚动，不触发左滑
  • 平台限制：仅 platform=lumi（内部票券）的票券支持左滑操作
  • 互斥：同时只允许一个卡片处于展开状态

可用操作

列表左滑操作说明
场次列表撤销场次开场前 3h+ 可撤销，调用 cancelGameRoundOrder
票券列表退票调用 refundCouponWithPayOrder，不可撤销
联系客服

拨打电话

用户中心提供「联系客服」按钮，点击后弹出操作面板：

  • 已选门店：拨打当前门店的客服电话（placeLists[storeId].phone）
  • 未选门店：拨打默认客服电话（service_phone）
  • 无号码：提示「无客服信息」
  • 通过 wx.makePhoneCall 发起通话
