场次排期与状态

数据加载

调用 /mz-admin/getGameRoundsOrdering，传入 date（日期），返回：

字段说明
game_round_lists按厅 ID 分组的场次列表（每个场次含时间、人数、角色、状态等）
store_rooms按门店 ID 分组的厅列表（A 厅、B 厅等）

场次状态流转

status 0 — 未开始 可操作

可签到、创角、编辑角色名/设备号。若已有人预约显示「已预约」，否则显示空白。

status 1/2 — 转后厅 进行中

已核销票券并转入后厅体验，不可再编辑角色。

status 3/4 — 游戏中 进行中

游戏进程已启动，玩家正在体验中。

status 5 — 已完成 结束

游戏已结束，场次归档。

status 6 — 已过期 过期

预约时间已过但未核销，场次自动过期。

A/B 厅切换

场次按厅（store_room）分组展示。顶部显示各厅选项卡，点击切换查看不同厅的场次排期。当前活跃厅索引存储在 activeDateIndex。

智能滚动定位

加载场次列表后，自动根据当前时间滚动到最近的场次位置。计算逻辑：将所有场次开始时间转换为分钟数，找到最后一个不超过当前时间的场次，页面自动 ScrollTo 该位置。仅在查看「今天」的日期时启用。

角色管理

展开场次详情

点击场次行展开该场次下的所有角色（订单）。每个角色展示：

字段说明
nickname角色昵称
gender性别（0=女，1=男）
device_id绑定的设备编号
is_checked_in是否已签到
group_name所属团队名
actived_phone激活者手机号
order_no订单编号

创建角色（形象弹窗）

点击未创建的角色（status=0），弹出形象选择弹窗：

  • 选择性别形象：左/右箭头切换男女角色形象图
  • 填写角色名称：手动输入或点击随机生成（如「聪慧的小狐狸」「勇敢的小老虎」等）
  • 填写设备编号：可选填，支持一键标记为「无」
  • 提交：调用 /mz-admin/updateGameRoleInfo，传入 role_id、nickname、gender
  • 角色名中文不超过 6 个字符

场次统计指标

每个场次行显示以下汇总数据：

页面顶部汇总
  • 已预约场次数 / 总场次数
  • 已预约人数 / 总可容纳人数
每场次的统计
  • 预约人数（bookedNum）
  • 已创建角色数（roleNum）
  • 已签到人数（checkedNum）
  • 已绑定设备数（deviceNum）
签到与电话联系

手动签到

管理员在场次详情中点击签到按钮：

  • 仅对 is_checked_in = false 且 status = 0 的角色生效
  • 弹出确认框「是否对此订单进行签到」
  • 调用 /mz-admin/makeOrderCheckIn，传入 order_id
  • 签到成功后刷新场次列表

电话联系

管理员可直接拨打用户电话：

  • 如果角色已有 actived_phone，直接拨打
  • 否则调用 /mz-admin/getPhoneByOrder，传入 order_id 获取手机号
  • 获取后缓存到 orderPhone 对象中，避免重复请求
  • 使用 wx.makePhoneCall 发起电话
