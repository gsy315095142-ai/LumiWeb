数据统计

数据统计模块为门店运营提供全面的数据洞察，包括票券汇总统计、各渠道/套餐的分项统计、退票和过期数据追踪，以及 Excel 对账单导出功能。

功能总览

页面结构

数据统计页面（/pages/stat/stat）由以下区域组成：

  • 页面标题：「数据统计」
  • 筛选条件区：日期范围选择器 + 门店下拉选择器
  • 导出按钮：导出对账单 Excel（仅超级管理员可见）
  • 汇总统计卡片：展示整体票券数据概览
  • 分项统计卡片：按渠道/类型分别展示详细数据
  • 底部导航栏：管理端 TabBar（ticket-bar 组件）

权限控制

通过 getUserPermissions('stat') 检查统计模块权限：

权限 Key说明影响范围
view-stat-hotel查看跨门店统计 + 切换门店 + 导出对账单可见门店下拉选择器和导出按钮；可查所有门店数据
无此权限仅查看当前门店数据门店选择器隐藏；store_id 自动使用 currentStoreId
筛选条件

日期范围选择（search-date 组件）

使用自定义 search-date 组件选择日期范围：

  • 日期索引：dataIndex=3，对应组件内的第 4 个快捷选项（通常为「近7天」或自定义）
  • 选择后触发：bind:search → onSearch(e)，获取 ticketDates 数组（[开始日期, 结束日期]）
  • 最大日期：pickerEnd 设置为当天，不可选择未来日期
  • 选择日期后自动触发数据加载（handleGetStatData）

门店选择

仅拥有 view-stat-hotel 权限的用户可见：

  • 数据来源：本地存储 placeLists（静默登录时写入）
  • 额外选项：列表头部插入「当前选择：全部」（id: 0）
  • 切换门店：onHotelChange → handleGetStatData，切换后自动刷新统计数据
  • 无权限时：隐藏门店选择器，store_id 自动使用当前门店 ID

初始化等待

页面加载时轮询检查 placeLists（每 100ms），确保门店数据已从登录接口写入本地存储后才初始化选择器。门店列表为空时，数据加载函数也会延迟 100ms 后自动重试。

分项统计卡片（newStatsData）

分项统计概览

汇总卡片下方，后端返回 newStatsData 数组，每项代表一个票券渠道或类型的独立统计卡片。

每张卡片的通用字段

字段类型说明

ticketNumNumber票券数量（张）
ticketValueNumber票券对应人数（人）
expandedBoolean套餐展开/收起状态

动态指标项（按条件渲染）

每张分项卡片下方可展示多种统计指标，根据后端是否返回该字段决定是否显示（wx:if）：

字段说明

receiveNoOrderNum已领取但未下单数量
refundNum退票数量
overdueNum过期数量
qrcodeNum二维码券码数量
qrcodeInvalidNum已失效二维码券码数量
packageRefundNum套餐退票数量（数值 > 100000 时字号缩小）
packageNum套餐数量

套餐展开/收起（packages）

若分项卡片包含 packages 数组，底部显示展开/收起按钮：

  • 收起状态：仅显示「展开」按钮 + 下箭头图标
  • 展开状态：列出每个套餐的名称、数量、人数
  • 无数据：展开后显示「暂无数据」
  • 切换：handleShowPackage 翻转 expanded 状态，箭头图标 180° 旋转动画

大数字适配

当 packageRefundNum.value > 100000 时，数字自动应用缩小样式（statistics-num-mini），防止溢出卡片。

数据加载流程

加载触发时机

  • 选择日期范围后：onSearch → 设置 ticketDates → handleGetStatData
  • 切换门店后：onHotelChange → handleGetStatData

handleGetStatData 流程

前置校验

门店列表不为空（否则 100ms 后重试）、日期范围已选择（ticketDates.length ≥ 2）、非加载中（loading = false）。

确定门店 ID

有 view-stat-hotel 权限 → 使用选择器中的门店 ID；无权限 → 使用 currentStoreId。

调用接口

调用 /mz-admin/getStoreDataStats，传入 date_start、date_end、store_id。

渲染数据

成功（code=0）→ 设置 totalStatData 和 newStatsData；失败 → Toast 提示后端 message。

空状态提示

数据未加载时（dataLoaded = false）显示占位区域：

  • 「数据未加载」
  • 根据权限提示「请选择日期」或「请选择日期和酒店」
  • 加载数据后（dataLoaded = true）切换为统计卡片列表
导出对账单

显示条件

导出按钮仅在以下条件同时满足时显示：

  • 有 view-stat-hotel 权限（超级管理员）
  • 选中了具体门店（selectedHotel > 0，「全部」选项 index=0 时不显示）

导出流程

前置校验

检查 ticketDates.length ≥ 2，否则提示「请先选择查询日期」。

构建下载 URL

/mz-admin/downloadStatements?date_start={start}&date_end={end}&store_id={id}

下载文件

调用 wxFileDownload(fileUrl, fileName)，文件名格式：{门店名}对账单-{开始日期}至{结束日期}.xlsx
