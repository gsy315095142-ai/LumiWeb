## 设计目的

本方案围绕对战体验中的**感知反馈**与**战术决策**两条主线展开：

| 方向 | 说明 |
|------|------|
| **玩家反馈** | 强化头显侧的命中、受击与得分表现，让 VR 玩家每一拳的结果清晰可感，提升对战临场反馈 |
| **观赛反馈** | 丰富投屏侧的文字、动画与数值联动，帮助观众快速读懂战局变化，增强观赛代入感 |
| **战术决策** | 通过部位得分差异、护盾消耗与回能规则，引导玩家在「打头 / 打身」「进攻 / 防守」之间做出更有意识、更有回报的取舍 |

## 命中得分规则

> 以下为本文采用的计分规则，便于对照「何时该出反馈」。

### 单次得分

| 命中类型 | 单次得分 |
|----------|----------|
| **头部**（头·左 / 头·右 / 头侧·左 / 头侧·右，任一） | **4** |
| **非头部**（胸、肋等其他部位） | **1** |

### 计分与扣血

| 项 | 规则 |
|----|------|
| 攻击方得分 | 每成功命中一次，阵营得分 **+ 本次得分** |
| 受击方扣血 | 当前 HP **- 本次得分**（不低于 0） |
| 大屏 / HUD 显示 | 展示双方 **剩余 HP**（初始满血，随受击递减） |
| 初始护盾 | **6 点** |
| 格挡消耗（非头部） | 成功格挡非头部攻击，防守方护盾 **-1** |
| 格挡消耗（头部） | 成功格挡头部攻击，防守方护盾 **-0.5** |
| 非头部命中回盾 | 攻击方成功命中非头部（未被格挡），攻击方护盾 **+1** |
| 连招第三拳回盾 | 完成连招第三拳，攻击方护盾 **+3** |

### 得分示例

| 命中情况 | 基础得分 | 连招第三拳 | 合计得分 / 扣血 | 护盾变动 |
|----------|----------|------------|-----------------|----------|
| 头部 | 4 | 否 | **4** | 无（命中） / 防守方 **-0.5**（格挡） |
| 非头部 | 1 | 否 | **1** | 攻击方 **+1**（命中） / 防守方 **-1**（格挡） |
| 头部 | 4 | 是 | **14** | 攻击方 **+3**（连招第三拳） |
| 非头部 | 1 | 是 | **11** | 攻击方 **+1**（命中）+ **+3**（连招第三拳） |
| 任意部位 | — | 被格挡 | **0** | 见上表格挡消耗 |

### 摆拳默认落点（手势触发时）

| 拳法 | 默认命中 | 单次得分 |
|------|----------|----------|
| 左摆拳 | 肋·右 | 1（非头部） |
| 右摆拳 | 肋·左 | 1（非头部） |

> 直拳以拳套实际碰到的打击面为准；摆拳在无碰撞落点时按上表默认部位计分。

### 命中得分时的反馈

攻击方命中敌方并获得分数时，头显与投屏提供联动反馈（具体样式、时长由美术与程序在实现阶段确认）。**得分数字跳出**与**血条扣除动画**在表现上同步，形成一条连续的受击反馈链。

| 端 | 触发 | 表现流程 | 范围 |
|----|------|----------|------|
| **投屏** | 命中计分成功（未被格挡） | ① 受击方头上弹出本次得分数字 → ② 数字自头上位置飞行至受击方血条 → ③ 抵达后播放血条扣除响应动画（HP 缩短 / 闪烁等） | **仅受击方一侧**；攻击方 HUD 不参与扣血动效 |
| **头显** | 同上 | 与投屏同步联动：受击方 HUD 上得分数字跳出 → 飞行至血条 → 血条扣除响应动画 | 同上 |

**作用：** 强调「这一拳打了多少分」以及「对方血量被削掉」；数字弹出与血条扣除在视觉上连贯一体，避免两段反馈脱节。
