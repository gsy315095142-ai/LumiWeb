### 8. 竞猜下注规则改为「只能下一次，只能押一方」

**新规则**：每场比赛用户只能下注一次，只能选红方或蓝方其中一方。下注后不允许追加或修改金额。

**下注后的 UI**：

| 下注前 | 下注后 |
|---|---|
| 选边按钮（红/蓝） | 隐藏 |
| 金额按钮（10/50/100/200/自定义） | 隐藏 |
| 「确认下注」按钮 | 隐藏 |
| — | 显示「✅ 已下注 🔴 战神阿飞」 |
| — | 显示「预期收益：XXX 💰」（= 下注金额 × 赔率） |
| — | 显示「取消下注」按钮 |

取消下注后恢复下注前状态。比赛结束后该场状态清空。

|---|---|---|
| 客户端 | `客户端-竞猜.js` | `matchState` 加 `placed:bool`；`renderBettingMatches()` 根据 `placed` 切两种 UI；新增 `cancelBet(mid)` |

### A2. 场次管理：厅卡片直接展开操作内容

现在的流程是：场次管理页 → 看到厅卡片 → 点击卡片 → 进入详情页才能操作。改成场次管理页的厅卡片直接展开显示当前步骤的操作内容，不再需要点进去。

**也就是说**：三个厅卡片（疾速冰球厅 / 雷霆击剑厅 / 烈焰拳王厅）每张卡片里就地渲染 `renderStep()` 的内容。如果某个厅正处于「报名管理」步骤，卡片里就能直接看到报名队列、选红蓝方按钮、开启竞猜按钮等，一步操作到位。

**不需要的内容**：`page-matchDetail` 页面可以删掉，`openMatchDetail()` 跳转逻辑也去掉。所有操作都在场次管理页内就地完成。

| 文件 | 改法 |
|---|---|
| `管理员端.html` | 删掉 `page-matchDetail`；`page-adminMatches` 内 `#zonesList` 改为渲染展开的厅卡片 |
| `管理员端-区域.js` | `renderZones()` 改为对每个厅调用 `renderStep()` 原地渲染步骤内容；删掉 `openMatchDetail()`、`stepPrev()` 的上一步逻辑（直接在场次管理页内完成全部流程） |

### A3. 管理 Tab 按钮顺序调整 + 新增「发放货币」

管理 Tab 底部按钮顺序从上到下调整为：

```
📲 发放货币    （新增）
📷 场次管理
📦 兑换物品
📋 账单管理
```

**「发放货币」页面流程**：

点击 `📲 发放货币` Tab → 进入扫码页面，主体是扫码框 + 「扫描用户身份码」按钮 → 扫到用户后，页面展示该用户信息（头像、昵称、💰 游戏币余额、💎 兑换币余额）→ 下方出现发放金额选择区域，样式照搬竞猜下注的金额选择：

```
[100] [200] [500] [1000] [自定义]
```

→ 选择发放理由（下拉：酒店入住 / 购买套餐 / 活动奖励 / 签到奖励 / 新手福利 / 补偿发放 / 其他）→ 补充说明（选填）→ 点击「确认发放」→ Toast 提示成功。

本质上是把现在 `管理员端-扫码.js` 里 `simulateScan → renderGiveContent` 那条链路，从原来的「先扫码再选操作类型」变成「点 Tab 直接进入扫码→发放」的专属流程，去掉中间选操作的步骤。

| 文件 | 改法 |
|---|---|
| `管理员端.html` | Tab 栏按钮顺序调整；新增 `page-adminGiveCoin` 页面（扫码框 + 用户信息区 + 金额选择 + 理由下拉 + 确认按钮） |
| `管理员端.js` | Tab 切换映射调整；新增 `switchTab('adminGiveCoin')` |
