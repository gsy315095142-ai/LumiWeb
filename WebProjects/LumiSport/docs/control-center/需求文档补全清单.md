## 一、全局性差异：6 步流程 vs 4 态代码

### 文档定义的 6 步流程

```
玩法配置 → 等待报名 → 报名中 → 竞猜中 → 比赛中 → 已结算
```

### 代码实际的 4 个 SessionStatus

| SessionStatus | 含义 | 对应文档中的哪几步 |
|---|---|---|
| `waiting` | 等待报名（含玩法配置） | 「玩法配置」+「等待报名」 |
| `betting` | 报名中（含竞猜开放） | 「报名中」+「竞猜中」 |
| `started` | 比赛进行中 | 「比赛中」 |
| `settled` | 已结算 | 「已结算」 |

### 差距说明

- **「玩法配置」和「等待报名」**：代码中合并为 `waiting`，无独立子状态区分。区分点为管理员是否已点过「确认玩法」→「开始报名」。
- **「报名中」和「竞猜中」**：代码中合并为 `betting`，通过 `guessingOpen` 布尔字段区分。管理员在报名中阶段手动点「开启竞猜」后才开放观众下注。
- **建议**：文档保持 6 步流程定义不变，在各页面中标注哪些子状态在代码中尚未独立实现，作为后续开发参考。

### 涉及页面

- `page-event.html`（赛事流程总览）：在流程图中标注开发状态
- `page-home.html`（首页：赛事选择）：在管理员场次管理流程进度条处标注

### 🎰 page-bet.html — 竞猜：参与押注

> 本页面缺失内容最多，是重点补全对象。

| 序号 | 缺失内容 | 详细说明 |
|---|---|---|
| W-1 | **实力悬殊规则（80% 阈值）** | 管理员点击「开启竞猜」的前置条件：①红蓝双方均已就位（redUserId/blueUserId 非空）；②任一方预期胜率不超过 80%（`START_MATCH_VOID_MAX_WIN_RATE = 0.8`）。不满足条件时后端返回错误：`当前对阵实力悬殊过大（任一方参考预期胜率高于 80%），暂不开放竞猜下注` |
| W-2 | **「取消本局竞猜」功能** | 仅在报名中（betting）阶段可用。执行后全额退还所有观众下注，局数不变但状态不变。文档中需确认是否已体现此按钮及其交互说明 |
| H-5 | **管理员操作的退款连锁规则** | ①「取消竞猜」：若当前为报名中或比赛中，先全额退还所有观众押注再收起场次；②「剔除选手」：剔除后 `guessingOpen` 自动重置为 false，若该场已有押注则全部退还 |
| J-3 | **已参赛不可竞猜的范围** | 不仅正式选手不能下胜负盘，**待审核状态的选手**（`myPendingSide` 非空）同样禁止下注。代码中 `mySide` 和 `myPendingSide` 任一非空都阻止竞猜 |
| B-1 | **赔率机制完整说明** | ①基于双方该玩法积分计算期望胜率：`myRating / (myRating + enemyRating)`；②扣除 10% 手续费：`odds = (1 / winRate) × (1 - 0.1)`；③赔率钳制在 **1.1 ~ 2.0** 之间；④报名/竞猜阶段赔率实时浮动，开赛瞬间锁定为 `lockedOdds` |
| B-2 | **各玩法固定赔率** | 胜负盘：1.1~2.0 浮动（取决于双方积分）；精准总分：固定 **20x**；精准分差：固定 **15x**；元素之王：固定 **3x**；躺平之王：固定 **3x** |
| B-3 | **下注最低金额** | 单次最低 **10 竞猜币**（`BET_MIN_AMOUNT = 10`） |
| B-4 | **每日签到系统** | 每日签到奖励 **100 竞猜币**（`DAILY_CHECK_IN_COINS = 100`），按 Asia/Shanghai 自然日限一次，新用户初始竞猜币为 **0** |
| B-5 | **预言家段位表** | 按累计竞猜派奖（`totalBettingWinnings`）判定，7 个段位如下：新手预言家 🔮（0+）、青铜预言家 🥉（2500+）、白银预言家 🥈（5000+）、黄金预言家 🥇（10000+）、钻石预言家 💎（20000+）、铂金预言家 👑（50000+）、王者预言家 🏆（100000+） |
| B-6 | **不同阶段的提示文案** | 报名未齐人时：`当前为报名阶段；待红蓝双方选手均通过审核就位后，将开放观众竞猜`。双方已就位但管理员未开竞猜：`双方选手已定；请管理员在「场次管理」流程操作中点击「开启竞猜」后，观众方可下注`。实力悬殊：`红蓝均已就位，但实力悬殊过大（参考：任一侧预期胜率高于 80%），本场暂不开放竞猜下注`。竞猜已开放：`竞猜已开放：下注后请以开赛结果为准（若临场不齐人等，开赛时仍可流局作废并退还）` |
| B-7 | **下注类型覆盖机制** | 同一类型（如胜负盘）重复下注会**覆盖**之前的金额和选项，不是叠加。取消下注则全额退还该类型 |
| M-1 | **开赛流局规则** | 开赛（`adminStartMatch`）瞬间检测：①若任一方无选手（redUserId/blueUserId 为空）；②或任一方预期胜率超过 80%。满足任一条件则：所有已下注押注**作废并全额退还**，赔率置为 null，`betsVoidedAtMatchStart = true`。竞猜区顶部展示通知：`开赛时未满足条件（未齐人或实力悬殊），本场押注已作废，竞猜币已全额退还` |

## 六、竞猜阶段提示文案汇总

| 阶段场景 | 提示文案 | 常量名 |
|---|---|---|
| 报名中，选手未齐人 | 当前为报名阶段；待红蓝双方选手均通过审核就位后，将开放观众竞猜 | `PRE_REGISTRATION_PHASE_HINT` |
| 双方已就位，管理员未开竞猜 | 双方选手已定；请管理员在「场次管理」流程操作中点击「开启竞猜」后，观众方可下注 | `ADMIN_GUESSING_NOT_OPEN_HINT` |
| 实力悬殊（>80%） | 红蓝均已就位，但实力悬殊过大（参考：任一侧预期胜率高于 80%），本场暂不开放竞猜下注 | `LOPSIDED_BETTING_PAUSED_HINT` |
| 竞猜已开放 | 竞猜已开放：下注后请以开赛结果为准（若临场不齐人等，开赛时仍可流局作废并退还） | `PRE_START_BETTING_HINT` |
| 管理员未开启竞猜 | 管理员尚未开启竞猜；请在管理人员开启后再下注 | `BETTING_GUESSING_CLOSED_MESSAGE` |
| 开赛流局退款 | 开赛时未满足条件（未齐人或实力悬殊），本场押注已作废，竞猜币已全额退还 | `MATCH_START_VOID_BANNER_TEXT` |
