📖页面定位

「胜负结算」是赛事进入已结算（settled）状态后的收尾环节。比赛结束后，Server 端自动将比分及附加数据写入场次，系统自动对竞猜玩法进行赔付。结算完成后，选手成绩归档到积分榜，管理员可选择「再来一局」或「返回首页」。

参数值常量说明

变动倍率20REF_SCORE_RATE决定单场变动幅度

🧮 积分结算公式

/* 已知量 */

R红 = 红方赛前积分  |  R蓝 = 蓝方赛前积分

S = R红 + R蓝   // 双方积分之和

/* 步骤一：算双方变化量 Δ — 由对方积分占比决定 */

Δ红 = 20 × R蓝 / S

Δ蓝 = 20 × R红 / S

/* 步骤二：应用到赛前积分 — 胜方加、败方减 */

红方胜 → R红 += Δ红 , R蓝 −= Δ蓝

蓝方胜 → R红 −= Δ红 , R蓝 += Δ蓝

平局 → 双方积分不变

💡 公式直觉

强者胜弱 → 加分少（败方积分低 → Δ 小）。弱者胜强 → 加分多（败方积分高 → Δ 大）。双方积分越接近，单场变动越大（最大接近 10）。

⚠️ 非零和：总积分通胀

Δ红 + Δ蓝 = 20 恒定，但胜方加对方占比 Δ、败方减对方占比 Δ——不对称叠加导致长期总积分膨胀。以下模拟直观感受。

📊竞猜玩法 · 结算对照

⚔️
胜负盘
依据：红 vs 蓝得分
猜中：下注额 × 锁定赔率
平局：全额退款
🔥
元素之王
依据：冰/火/风球最多者
并列：ice > fire > wind
猜中：下注额 × 3 倍
🥊
躺平之王
依据：totalKnockdowns
猜中：下注额 × 3 倍
🎯
精准总分
依据：红方得分 + 蓝方得分
猜中：下注额 × 20 倍
📏
精准分差
依据：|红 − 蓝|
猜中：下注额 × 15 倍
