🔑 Token 持久化与 JWT 鉴权
登录成功后，后端签发 JWT 令牌（有效期默认 7 天），前端将 accessToken、userInfo（含 role）和 managedStores（门店列表）分别持久化到微信小程序 Storage。后续请求自动携带 Authorization: Bearer <token>。
// JWT Payload 结构
{
  "sub": 1,                    // 用户 ID
  "role": "user" | "admin",     // 角色（白名单判定）
  "storeIds": [1, 3],         // 管理的门店 ID（仅 admin）
  "mp_openid": "oX1234...",   // 微信 OpenID
  "iat": 1710000000,         // 签发时间
  "exp": 1710086400         // 过期时间
}
💾 本地存储键值
键名内容说明
tokenJWT accessToken用于请求鉴权，Bearer Token
userInfo{ id, nickname, role, mp_openid }用户基本信息，role 由白名单判定
managedStores[{ id, name }]管理的门店列表，仅 admin 有值
currentStoreIdnumber当前选中的门店 ID
💾
写入时机：登录成功后
setToken → Storage('token')
setUserInfo → Storage('userInfo') { id, nickname, role, mp_openid }
setManagedStores → Storage('managedStores') [{ id, name }]
🔍
校验时机：每次打开小程序
App.vue → userStore.checkAutoLogin() → fetchProfile()
成功：恢复登录态 + 刷新白名单信息  |  失败（401）：清除 token + 跳回登录页
🗑️
清除时机：退出登录 / token 失效
userStore.logout() → clearAuth() → 移除 token + userInfo + managedStores
→ reLaunchSafe('/pages/login/index')
