🔌后端接口

弹幕模块位于后端 modules/danmaku，提供两个 RESTful 接口，均需 JWT 认证：

📥 GET /api/danmaku

按场次增量拉取弹幕
参数：sessionId、afterId（默认 0）、limit（默认 50，最大 100）

📤 POST /api/danmaku

发送弹幕
参数：sessionId（int, ≥1）、content（string, 1~80字）

📤 发送弹幕校验链

后端收到 POST 请求后依次执行以下校验，任一不通过即返回错误：

步骤校验内容错误提示
① 空内容校验 content.trim() 为空 → 拒绝 弹幕内容不能为空
② 长度校验 content.length > 80 → 拒绝（DTO 层 @MaxLength(80)，服务层二次校验） 弹幕内容过长（最多 80 字）
③ 敏感词校验 匹配词库中的敏感词（子串匹配，大小写不敏感）→ 拒绝 弹幕内容包含敏感词，请修改后重试
④ 场次存在校验 sessionId 对应的场次不存在 → 拒绝 场次不存在
⑤ 用户存在校验 JWT 中的 userId 对应的用户不存在 → 拒绝 用户不存在

📤 响应结构

DanmakuMessagePayload

id: number — 弹幕唯一 ID
sessionId: number — 所属场次 ID
userId: number — 发送者用户 ID
displayName: string — 发送者昵称
content: string — 弹幕正文
createdAt: string — 发送时间（ISO 8601）

DanmakuListPayload（GET 响应）

messages: DanmakuMessagePayload[] — 弹幕列表（时间正序）
maxId: number — 当前列表中最大 ID，客户端用于下次轮询的 afterId
